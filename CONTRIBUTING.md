# :scroll: Contribution Guidelines

## What belongs on this list

This is a **curated** list, not a comprehensive catalog. Tools should be:

- **Useful for forensic work** — not just tangentially related to security
- **Documented** — has a README with clear usage instructions
- **Maintained** — shows signs of active development or a stable, complete state
- **Tested in practice** — has been used in real forensic work, not just a proof of concept

Pull requests that don't meet these criteria may be closed.

## Formatting

- Use the following format: `[Item Name](homepage link) - Description.`
- Clear, concise descriptions for each link, followed by a period
- Lists within each section are alphabetized
- Add sections if necessary, use existing sections if possible
- No duplication of tools — put them where they make the most sense
- One tool per pull request

## Before you submit

- Search the list first to make sure the tool isn't already included
- Check that the link works and points to the right place
- Make sure your PR follows the formatting guidelines above

By submitting a pull request, you agree to release your submission under the [LICENSE](LICENSE).
